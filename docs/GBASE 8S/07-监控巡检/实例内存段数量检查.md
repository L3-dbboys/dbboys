执行以下命令检查数据库内存段：
```
onstat -g seg
```
观察是否存在多个【class】列为【V】状态的内存段，如存在，系统初始分配的内存已不足以满足业务需求，数据库已动态增长内存段。

【onstat -g seg】示例输出：

如以上输出中，数据库已分配5个【class】为【V】的内存段，表明数据库运行过程中，已动态扩展4个内存段，需要持续观察，数据库是否会持续扩展内存段，如持续扩展，可能存在内存泄漏的情况。
