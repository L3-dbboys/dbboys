### 示例1-ontape重定向恢复
操作步骤与其他恢复步骤一致，需指定rename和映射文件名称，如：
```
ontape -r -rename -f rename.txt
```
rename.txt里添加原数据文件路径和偏移量，对应新的数据文件和偏移量，如（需提前创建好重定向文件及权限，参考ontape异机恢复）：
```
/opt/gbase/dbs/rootdbschk001 0 /opt/gbase/dbs/rootdbschk002 0
/opt/gbase/dbs/plogdbschk001 0 /opt/gbase/dbs/plogdbschk002 0
/opt/gbase/dbs/llogdbschk001 0 /opt/gbase/dbs/llogdbschk002 0
/opt/gbase/dbs/tempdbs01chk001 0 /opt/gbase/dbs/tempdbs01chk002 0
/opt/gbase/dbs/sbspace01chk001 0 /opt/gbase/dbs/sbspace01chk002 0
/opt/gbase/dbs/datadbs01chk001 0 /opt/gbase/dbs/datadbs01chk002 0
```
onbar重定向恢复与ontape类似，在正常的恢复命令后增加-rename -f并准备好映射文件即可。
