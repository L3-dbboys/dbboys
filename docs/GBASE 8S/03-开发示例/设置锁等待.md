### 说明
数据库默认锁等待时间为0，一般需要设置锁等待时间，避免在发生锁冲突时出现报错，锁等待时间为会话级别，设置后会话全局有效，与事务无关。

### 示例1-库全局设置
连接到该库的所有连接默认等待该时间。
```
dbaccess gbasedb -<<!
create procedure public.sysdbopen()
set lock mode to wait 10;
end procedure;
!
```
创建以上存储过程后，连接到gbasedb的所有连接默认锁等待时间为10秒。

### 示例2-当前会话设置
```
dbaccess gbasedb -<<!
set lock mode to wait 10;
update ...
delete ...
!
```
在会话中执行set lock mode to wait 10，该会后后续所有操作锁等待时间为10秒。

### 示例3-在JDBC URL设置所有连接锁等待
```
URL_STRING = "jdbc:gbasedbt-sqli://192.168.1.1:9088/gbasedb:GBASEDBTSERVER=gbase01;IFX_LOCK_MODE_WAIT=10";
```
IFX_LOCK_MODE_WAIT=10,该URL的连接锁等待时间为10秒。

### 示例4-在GBase Data Studio中设置锁等待时间
在已创建的连接上右键--编辑连接--驱动属性--右键--添加新属性--名称填入IFX_LOCK_MODE_WAIT，值填入10。
