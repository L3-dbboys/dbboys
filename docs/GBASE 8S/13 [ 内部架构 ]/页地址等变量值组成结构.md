PAGE ADDRESS
```
page address结构：0xPPPPPPPPSSSS

|----------------PPPPPPPP----------------|--------SSSS--------|
|----------------- 4Bytes ---------------|------- 2Bytes -----|
|----------- offset in chunk ------------|------ chunknum ----|

#页地址位于每个页页头前6位，前四位为偏移量，后2位为chunk号
```
LOGPOS
```
logic log位置结构：0xPPPPPSSS

|----------PPPPP----------|------SSS------|
|-------- 2.5Bytes -------|--- 1.5Bytes --|
|--- offset in logfile ---| offset in page|

#逻辑日志地址用4个字节描述，前2.5字节描述数据页在逻辑日志文件中的偏移量，
#后3个字节描述地址在页里的偏移量
```
PARTNUMBER
```
partnum结构：0xPPPSSSSS

|------PPP------|----------SSSSS----------|
|--- 1.5Bytes --|-------- 2.5Bytes -------|
|---- dbsnum ---|-- logic offset in TBS --|

#partnum总长4字节，前1.5字节存储空间号，后2.5字节存储表描述页在TBLSpace中的逻辑位置
#如rootdbs中TBLSpace本身的描述页为TBLSpace TBLSpace里第一页（第0页为bitmap页），他的parnum为0x100001
#sysdatabases的partnum为第二页，partnum为0x100002
#如需打印某表的描述页 oncheck -pp 0xPPP 0xSSSSS 即可，打印页与oncheck -pt xxxdb:xxxtab一致
```
ROWID
```
rowid结构：0xPPPPPPSS

|-------------PPPPPP-----------|----SS----|
|------------ 3Bytes ----------|- 1Bytes -|
|- page logic offset in table -|-- slot --|

#rowid并非物理存储在数据页，而是计算出来行在表空间里的逻辑位置
#分片表由于具有多个partnum（表空间），故无法计算rowid，所以分片表默认无rowid
#创建分片表with rowids，每次insert rowid会加1，最多insert 2147483647次数据
```
