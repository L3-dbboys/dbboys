数据库默认隔离级别为COMMITTED READ（CR），一般不需要更改，隔离级别为会话级别，设置后会话全局有效，与事务无关。

隔离级别种类：
DIRTY READ;   --脏读（1）
COMMITTED READ;  --提交读（2）
CURSOR STABILITY;  --游标稳定（3）
REPEATABLE READ;   --可重复读（4）
COMMITTED READ LAST COMMITTED;  --最后提交读（5）

参考以下方式设置隔离级别：
1、设置某库全库隔离级别，连接到该库的所有连接默认为该隔离级别。
```
dbaccess testdb -<<!
create procedure public.sysdbopen()
SET ISOLATION TO DIRTY READ;  --设置为脏读
end procedure;
!
```
创建以上存储过程后，连接到testdb的所有连接默认隔离级别为脏读。

2、会话中执行sql设置锁隔离级别
```
dbaccess testdb -<<!
SET ISOLATION TO DIRTY READ;
update ...
delete ...
!
```
在会话中执行SET ISOLATION TO DIRTY READ，该会后后续所有操作锁隔离级别为脏读。

3、在JDBC URL中设置隔离级别
```
URL_STRING = "jdbc:gbasedbt-sqli://192.168.1.1:9088/gbasedb:GBASEDBTSERVER=gbase01;IFX_ISOLATION_LEVEL=1";
```
IFX_ISOLATION_LEVEL=1,该URL的连接隔离级别为脏读。

4、在GBase Data Studio中设置锁等待时间
在已创建的连接上右键--编辑连接--驱动属性--右键--添加新属性--名称填入IFX_ISOLATION_LEVEL，值填入需要设置的隔离级别数字。
