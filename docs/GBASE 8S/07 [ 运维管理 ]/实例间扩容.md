使用onspaces -a命令为数据库空间增加数据块（chunk），可参考以下语法：
```
onspaces { -a <spacename> -p <path> -o <offset> -s <size> [-m <path> <offset>]
               { { [-Mo <mdoffset>] [-Ms <mdsize>] } | -U }
               }
```
参数说明
```
-a	指定需要添加chunk的数据库空间
-p	数据块（chunk）路径
-o	数据块（chunk）偏移量
-s	空间大小（KB）
```
使用【gbasedbt】用户参考以下命令创建数据文件：
```
touch /data/gbase/datachk02
chmod 660 /data/gbase/datachk02
```
**注意：需要确认chunk路径的设备存在，要求属主为gbasedbt:gbasedbt，且权限为660，如果是数据库集群，需要在集群所有节点创建响应的数据文件。**

示例1：为数据库空间datadbs01增加一个1024000KB的chunk
```
onspaces -a datadbs01 -p /data/gbase/datachk02 -o 0 -s 1024000
```

**注意：如果是数据库集群，只需在集群主节点执行【onspaces】命令扩容，备机自动同步。**
