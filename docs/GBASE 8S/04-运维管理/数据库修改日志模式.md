### 修改数据库日志模式为NOLOG  
```
ondblog nolog mydb
```
返回0表示成功。
### 修改数据库日志模式为BUFFERED LOG  
```
ondblog buf mydb
```
返回0表示成功。
### 修改数据库日志模式为UNBUFFERED LOG  
```
ondblog unbuf mydb
```
返回0表示成功。

### 执行伪备份
数据库日志从nolog调整为buf或unbuf，需要对数据库执行备份才能使用。
```
onbar -b -F
```

**注意：如修改日志模式返回值不为0，执行onstat -g sql检查该数据库上是否还有连接，如有，使用onmode -z \<sid>杀掉连接。**
