### 空间扩容语法
使用onspaces -a命令为数据库空间增加数据块（chunk），可参考以下语法
```
onspaces { -a <spacename> -p <path> -o <offset> -s <size> [-m <path> <offset>]
               { { [-Mo <mdoffset>] [-Ms <mdsize>] } | -U }
               }
```
**注意：在扩容数据库空间之前，需要确认chunk路径的设备存在，要求属主为gbasedbt:gbasedbt，且权限为660。**
### 示例1-空间扩容
为数据库空间datadbs01增加一个1024000KB的chunk
```
touch /data/gbase/datadbs01chk002
chmod 660 /data/gbase/datadbs01chk002
onspaces -a datadbs01 -p /data/gbase/datadbs01chk002 -o 0 -s 1024000
```
其他类型数据库空间扩容与标准数据库空间扩容语法一致。  
**注意：如果是数据库集群，需要在主备均创建文件，但只需在集群主节点执行【onspaces】命令扩容，备机自动同步。**
