### 错误跟踪

确认参数DUMPSHMEM为0，避免dump内存占用大量磁盘空间
```
onstat -c |grep DUMPSHMEM
```
如该参数不为0，执行以下命令修改：
```
onmode -wf DUMPSHMEM=0
```

跟踪需要跟踪的错误号，如跟踪错误-211，在此后发生的211错误将被记录
```
onmode -I 211
```

检查$GBASEDBTDIR/tmp目录，如有af.xxxxxx生成，表示已捕捉到错误并生成了数据库运行状态快照。检查af.xxxxxx文件，检查发生错误的sql及会话、线程状态等信息。
